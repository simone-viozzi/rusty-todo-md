# Experiment: test-suite pruning vs snapshot suite (issue #202)

One-shot experiment, **not project infrastructure.** Lives here so the
data, the tooling that produced it, and the verdicts are all in one
place — and out of the repo root.

## What's tracked here

| File | Provenance | Purpose |
|---|---|---|
| `overlap-report.md` | **Generated** by `scripts/overlap_analysis.py` | Per-test line overlap with the snapshot test union; distribution histogram; snapshot-only `src/` coverage table; ranked tables. |
| `overlap-data.json` | **Generated** by `scripts/overlap_analysis.py` | Machine-readable form of the overlap report. Consumed by `build_subagent_prompts.py`. |
| `triage-verdicts.md` | **Hand-assembled** from subagent output during the implementation session | The 28 KEEP verdicts the Sonnet subagent returned (one per candidate ≥ 0.70). |
| `scripts/extract_per_test_coverage.sh` | Source | Builds the per-test JSON corpus into `coverage/per-test-json/`. |
| `scripts/overlap_analysis.py` | Source | Computes overlap, regenerates `overlap-report.md` and `overlap-data.json`. |
| `scripts/build_subagent_prompts.py` | Source | Generates the per-candidate Sonnet prompts (under `coverage/subagent-prompts/`, also not tracked). |

Everything written to `coverage/` is gitignored — that includes the
~55MB of per-test JSON, the merged profdata, and the prompt corpus.
Re-running the pipeline reproduces them.

## How to re-run

From the repo root, with `cargo-llvm-cov` + `llvm-tools-preview`
available:

```bash
# 1. Build instrumented test binaries (and the main binary that the
#    snapshot tests spawn as a subprocess).
cargo llvm-cov clean --workspace
eval "$(cargo llvm-cov show-env --sh)"
cargo test --no-run
cargo build --bin rusty-todo-md

# 2. Write coverage/binaries.list with the exact instrumented test
#    binaries you want to measure (paths like
#    target/debug/deps/<crate>-<hash>). Empty binaries (rusty-todo-md
#    bin/utils/mod) should be omitted.

# 3. Run per-test extraction. This writes
#    coverage/per-test-json/*.json, one per #[test].
bash docs/experiments/test-pruning-202/scripts/extract_per_test_coverage.sh

# 4. Compute the overlap report.
python3 docs/experiments/test-pruning-202/scripts/overlap_analysis.py

# 5. Generate per-candidate subagent prompts (one file per tests/*.rs
#    candidate with overlap >= 0.70).
python3 docs/experiments/test-pruning-202/scripts/build_subagent_prompts.py

# 6. Fan out subagents per the rubric in arch-review. Subagent verdicts
#    are not regenerated by this code; they were captured by hand into
#    triage-verdicts.md during the implementation session.
```

## Bottom line

28 `tests/*.rs` candidates reviewed, 0 DELETE, 28 KEEP. The snapshot
corpus alone covers **46.3% of `src/`** (574 / 1239 executable lines)
— everything else lives behind the integration tests. The deletion
pass is gated on snapshot-corpus expansion; see follow-up #205.
