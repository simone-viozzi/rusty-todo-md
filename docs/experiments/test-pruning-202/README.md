# Experiment: test-suite pruning vs snapshot suite (issue #202)

One-shot experiment, **not project infrastructure.** Lives here so the
data, the tooling that produced it, and the verdicts are all in one
place — and out of the repo root.

The first pass (snapshot corpus = 5 happy-path fixtures) measured a
46% snapshot-only `src/` coverage and returned 28 KEEP / 0 DELETE —
the corpus was too narrow to subsume anything. PR #207 then expanded
the snapshot fixture set to 23 (adding error paths, custom flags,
glob excludes, multi-language parsers, second-run merge, and conflict
markers). The pipeline was re-run against the expanded corpus and
**11 of 32 `tests/*.rs` candidates are deleted in this PR**.

## What's tracked here

| File | Provenance | Purpose |
|---|---|---|
| `overlap-report.md` | **Generated** by `scripts/overlap_analysis.py` | Per-test line overlap with the snapshot test union; distribution histogram; snapshot-only `src/` coverage table; ranked tables. |
| `overlap-data.json` | **Generated** by `scripts/overlap_analysis.py` | Machine-readable form of the overlap report. Consumed by `build_subagent_prompts.py`. |
| `triage-verdicts.md` | **Hand-assembled** from subagent output during the implementation session | DELETE/KEEP verdicts from the Sonnet subagents (one per candidate ≥ 0.70). |
| `scripts/extract_per_test_coverage.sh` | Source | Builds the per-test JSON corpus into `coverage/per-test-json/`. |
| `scripts/overlap_analysis.py` | Source | Computes overlap, regenerates `overlap-report.md` and `overlap-data.json`. |
| `scripts/build_subagent_prompts.py` | Source | Generates the per-candidate Sonnet prompts (under `coverage/subagent-prompts/`, also not tracked). Now data-driven: reads `tests/snapshot_tests.rs` + `tests/snapshots/*.snap` so the corpus digest tracks the live fixture set. |

Everything written to `coverage/` is gitignored — that includes the
~55MB of per-test JSON, the merged profdata, and the prompt corpus.
Re-running the pipeline reproduces them.

## How to re-run

From the repo root, with `cargo-llvm-cov` + `llvm-tools-preview`
available:

```bash
# 1. Build instrumented test binaries (and the main binary that the
#    snapshot tests spawn as a subprocess).
cargo llvm-cov clean --workspace
eval "$(cargo llvm-cov show-env --sh)"
cargo test --no-run
cargo build --bin rusty-todo-md

# 2. Write coverage/binaries.list with the exact instrumented test
#    binaries you want to measure (paths like
#    target/debug/deps/<crate>-<hash>). Empty binaries (rusty-todo-md
#    bin/utils/mod) should be omitted. The hashes are stable across
#    rebuilds with the same source, so the existing list usually
#    keeps working without edits.

# 3. Run per-test extraction. This writes
#    coverage/per-test-json/*.json, one per #[test].
bash docs/experiments/test-pruning-202/scripts/extract_per_test_coverage.sh

# 4. Compute the overlap report.
python3 docs/experiments/test-pruning-202/scripts/overlap_analysis.py

# 5. Generate per-candidate subagent prompts (one file per tests/*.rs
#    candidate with overlap >= 0.70). Each prompt now embeds the live
#    snapshot corpus inline — args, todo_md, stderr, git_index — so
#    the subagent can compare assertions directly.
python3 docs/experiments/test-pruning-202/scripts/build_subagent_prompts.py

# 6. Fan out subagents per the rubric. Subagent verdicts are not
#    regenerated by this code; they were captured by hand into
#    triage-verdicts.md during the implementation session.
```

## Bottom line

32 `tests/*.rs` candidates reviewed (overlap ≥ 0.70), **11 DELETE**,
**21 KEEP**. The 11 deleted tests are the ones the expanded snapshot
suite subsumes on both stdout and stderr; the 21 retained tests
assert on something snapshots still can't reach: stderr error text
on non-zero exit, internal-API invariants, git-index state beyond
`--auto-add`, multi-step second-run / file-removal scenarios with
custom flags, or merge-driver install/auto-install/self-heal flows.

Snapshot-only `src/` coverage rose from **46.3%** (5 fixtures) to
**64.9%** (23 fixtures) — the integration-only delta shrank from
40 points to 22, which is the slack the next fixture-expansion pass
would need to close before further deletions are safe.
